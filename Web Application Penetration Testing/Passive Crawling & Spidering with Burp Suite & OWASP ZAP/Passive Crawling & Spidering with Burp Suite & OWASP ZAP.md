
### Passive Crawling and Spidering are reconnaissance technique used in web application penetration testing to map and understand a web application without aggressively interacting with it.

This phase is extremely important because before attacking a web application, a pentester needs to discover:
- Pages
- Directories
- Parameters
- Forms
- APIs
- JavaScript files
- Authentication points
- Hidden functionality

Web applications work through client-server communication using HTTP requests and responses. During crawling/spidering, tools observe and follow these HTTP requests and responses to discover application content.

## What is Crawling?
Crawling means Visiting a web pages and collecting information about links,forms,script,parameter and resources.

Think of it like:
- A search engine bot exploring a website
- Following every discovered hyperlink
- Building a map of the application

Example:

You visit:

```
http://testphp.vulnweb.com
```

The crawler discovers:

```
/index.php/login.php/products.php/cart.php/admin/
```

Then it recursively follows those pages too.


# Simple Understanding

|Term|Meaning|
|---|---|
|Crawling|Manual/semi-manual exploration|
|Spidering|Automatic exploration|

---
# Crawling

You browse the website yourself.

Example:

- Open homepage
- Click login
- Open products
- Visit cart

Tools like:

- Burp Suite
- OWASP ZAP

observe your traffic and build a sitemap.

This is often called:

> Passive crawling

because the tool mainly watches traffic.

---
# Spidering

The tool itself automatically follows links.

You start the spider and it:

- clicks links automatically
- visits pages recursively
- maps the application

This is:

> Active crawling/spidering

# Example

## Manual Crawling

You manually visit:

```
//login/products/cart
```

Burp records them.

---

## Spidering

Burp/ZAP automatically discovers:

```
//login/products/cart/admin/api/uploads
```

without you manually opening each page.

# Passive Crawling

Passive crawling means:

> The tool observes traffic generated naturally by the user/browser.

The tool does NOT aggressively attack the site.

It simply watches:

- HTTP requests
- Responses
- URLs
- Parameters
- Cookies
- JavaScript files

This is safer and stealthier.

---

# Example of Passive Crawling

Suppose you manually browse:

```
Home → Login → Products → Cart
```

Burp Suite or ZAP records:

```
//login.php/products.php?id=1/cart.php/api/user
```

No extra requests are generated by the tool.

---

# Why Passive Crawling is Useful

Passive crawling helps discover:

- Hidden endpoints
- API routes
- Session cookies
- JavaScript files
- Parameters
- Technologies used
- Authentication flows

It is especially useful during:

- Bug bounty reconnaissance
- Internal pentests
- Stealth assessments
